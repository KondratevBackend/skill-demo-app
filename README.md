# skill-demo-app

**skill-demo-app** — это демонстрационный backend-проект, реализующий сервис подписки с управлением доступом через telegram бота.

Проект задуман не как “ещё один CRUD”, а как попытка показать, как можно строить **интеграционный backend**, где есть:
- внешние системы / интеграции (payment, tg, multi-API)
- асинхронные процессы
- разделение ответственности внутри монолита
- и контроль над сложностью по мере роста

Это не production-решение, а **песочница для архитектуры и практик**.

> Этот проект предназначен только для демонстрации, пожалуйста, не используйте его в незаконных целях и в производственной среде.

## Что внутри

Система организована как **распределённый монолит**: один кодбейс, но несколько независимых рантаймов.

- Telegram-бот — основной интерфейс (пользователь + админ)
- HTTP API (FastAPI) — точка входа для внешних событий (webhook'и)
- Worker \ Sheduler — обработка фоновых задач и периодические задачи

## Архитектурный подход

В основе — довольно простая, но строгая модель:

- API слой — принимает входящие данные
- Сервисный слой — содержит бизнес-логику
- Репозиторий — отвечает за работу с БД

Ключевые принципы:

- **никакой бизнес-логики в хендлерах**
- **явные зависимости через DI (punq)**
- **минимальная связанность между модулями**
- **контроль точек интеграции**
- **получение, преобразование и передача данных между слоями с использованием DTO**

Отдельный фокус — на том, чтобы код было легко расширять и переиспользовать, не переписывая существующий.
